Data minimization

Only provide information that is useful for the naming decision. NamEngine does not need sensitive personal data to generate names.

What not to submit

Do not submit Social Security numbers, payment card numbers, passwords, medical records, legal case details, private family secrets, or sensitive information about children.

Third-party processors

NamEngine may use hosting, AI, analytics, email, and payment providers to operate the product. These providers should be configured through server-side keys and environment variables, not exposed in public code.

Deletion requests

During beta, users should be able to request deletion of submitted naming data. The operator may need a session link, payment email, or other context to locate the relevant record.

Security practices before paid launch

  • Store API keys and payment keys in Render environment variables or equivalent secret storage.
  • Keep payment processing with Stripe or another payment processor.
  • Do not commit secrets or service-account JSON files to the repository.
  • Run mobile, desktop, payment, and legal-link smoke tests before deploy.

Beta limitation

This page describes intended beta practices. It should be reviewed and tightened before broader public launch or before serving users in jurisdictions with specific privacy requirements.
